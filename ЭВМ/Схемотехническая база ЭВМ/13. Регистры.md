**Регистр** — устройство построенное на триггерах, предназначенное для хранения n-разрядных двоичных чисел и выполнения преобразований.
  
Различают 3 типа регистров:
- Хранения. У каждого триггера свой независимый вход и выход. Используются для хранения многоразрядного числа.
- Сдвига. Цепочка триггерных схем, связанных цепями переноса.
- Универсальные. Объединяют в себе функции вышеуказанных.

Сдвиги:
### Логический
В отличие от того что подано на SLI_(Shift Input Left)_/SRI зависит то, чем заполняются пустые значения при сдвиге.

### Циклический
То что вышло с одного конца, попадает в другой.

Микрооперации регистра — список действий над ними.

## Регистр хранения

!!фотка

Синхронный регистр с асинхронным сбросом в 0. Динамическая синхронизация по переднему фронту. и разрешением чтения/записи.
### Распиновка

- CE — Chip Enable
- CLR _(clear)_ = Reset
### Синхронизация
#### Динамическая
 Треугольник углом вправо — синхронизация происходит по переднему фронту.
Треугольник углом влево — синхронизация происходит по заднему фронту.

#### Статическая
**Статическая** — если происходит по уровню.
Такой регистр называется защелкой.

## Сдвиговый регистр

- L _(load)_: 1 — загрузка, 0 — сдвиг
  Это выбор микроперации

## Счетчик

Устройство последовательностного типа, предназначенное для подсчета числа входных импульсов и хранения результатов.

Каждый новый импульс меняет состояние счетчика на следующее.


- **Модуль счета** — число различимых состояний. После достижения максимального сбрасывается в 0.
- Входные импульсы поступают на **счетный вход**.
- Каждое состояние определяется только предыдущим.
