
## Логическая схема

**Логическая схема** — это совокупность структурных логических элементов
определенного типа и структура (схема) их соединения.

Схема соединения должна обладать следующими свойствами:

1. Имеется ровно один элемент, выход которого не соединен ни с одним входом
(выход логической схемы).
2. Имеется ровно n входов, которые не соединены ни с одним выходом. Каждому
входу сопоставляется логическая переменная.
3. Схема не должна содержать обратные связи.

Точка соединения выхода элемента с входом называется узлом схемы.

### Анализ логической схемы

Решение задачи анализа состоит в последовательном восстановлении
функций, вычисляемых в узлах схемы, при перемещении от входов к
выходу схемы.

Если определены функции на входах структурного элемента, то функция
на выходе элемента определяется как суперпозиция функции,
вычисляемой элементом, и входных функций.

## Задача SAT (Задача выполнимости булевых формул)

**ФОРМУЛИРОВКА:** задана некоторая формула $\phi$ от $n$ переменных. Существует ли набор значений для всех переменных в этой формуле, при котором $\phi$ = 1?

Задача нетривиальна при больших $n$.

Имеющиеся алгортмы решений:
- DPLL (Davis-Puthman-Logeman-Loveland) — _алгоритм Дэвиса — Патнема — Логемана — Лавленда_
- CDCL (Confilct-Driven Clause Learning)

### SAT-решатели 

Специализированная компьютерная программа, которая принимает на вход булеву формулу в формате КНФ и пытается определить её выполнимость, называется SAT-решателем.

SAT-решатели

Для чего нужно:
- Формальная верификация программного и аппаратного обеспечения
- Криптоанализ
- Решение задач планирования и оптимизации
- Реверс-инжиниринг

Общий подход к решению:
- Описание задачи (спецификация, схема, код и т.д.) переводится в гигантскую улеву формулу
- Полученная формула подается на вход SAT-решателю
- Полученный результат интерпретируется в зависимости от задачи.

### Задачи полиноминальной сложности (класс P)

Задача называется **полиномиальной**, т.е. относится к классу P, если существует константа k и алгоритм, решающий задачу, имеющий сложность $O(n^{k})$, где n - длина входа алгоритма в битах.

Задачи класса P – это, интуитивно, задачи, решаемые за реальное время («практически разрешимые задачи).

Преимущества алгоритмов P-класса:
- Для большинства задач из класса P константа k меньше 6
- Класс P обладает свойством естественной замкнутости (сумма или произведение полиномов есть полином)
  
###  Полиномиально проверяемые задачи (класс NP)

- Класс NP — задачи, которые могут быть решены за полиномиально
выраженное время с помощью недетерминированной
вычислительной машины, то есть машины, следующее состояние
которой не всегда однозначно определяется предыдущими.
- Работу такой машины можно представить как разветвляющийся на
каждой неоднозначности процесс. Задача считается решённой, если
хотя бы одна ветвь процесса пришла к ответу.

**Альтернативное определение**: к классу NP относятся задачи, решение которых мы можем проверить за **полиномиальное** время с помощью **дополнительной** информации полиномиальной длины, данной нам свыше. (эта информация называется _оракул_/_сертификат_/_свидетель_)

### Проблема N = NP (проблема перебора)

**Можно ли все задачи, решение которых проверяется с полиномиальной сложностью, решить за полиномиальное время?**

  
$f \in P \implies f \in NP$
  т.к. проверкой за полиноминальное время может быть просто решение задачи.


### NP-полные задачи (класс NPC)

$NPC \subset NP$
Любая $NP$-задача может быть сведена к $NPC$
Если существует задача, принадлежащая классу NPC, для которой существует
полиномиальный алгоритм решения, то класс P совпадает с классом NP, т.е. P=NP

Найдено много $NPC$-задач, но пока НЕ найдено для них ни одного полиномиального алгоритма решения.
